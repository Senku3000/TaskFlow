# TaskFlow

A full stack task management platform built for professors - combining a Kanban board with AI powered syllabus parsing and ML driven student grade prediction.


## Features

- **Kanban Board** - Drag and drop task management with Backlog, In Progress, Review, and Done columns with configurable WIP limits
- **AI Syllabus Parsing** - Upload a PDF syllabus and automatically generate tasks using Groq's LLaMA 3.3 70B model via LangChain
- **ML Grade Prediction** - Predict final end-semester grades from CIA (Continuous Internal Assessment) scores using a custom-trained scikit-learn model
- **Multi-Class Management** - Professors can create and switch between multiple classes, each with its own board and student roster
- **Student Analytics Dashboard** - View, add, edit, and delete students with color-coded grade visualization
- **Secure Authentication** - Session-based auth with Werkzeug password hashing and per-professor data isolation
- **Import/Export** - Backup and restore board data as JSON
- **Responsive UI** - Dark-themed interface that works on desktop and mobile

## Tech Stack

| Layer | Technology |
|-------|-----------|
| **Backend** | Python, Flask, PyMySQL |
| **Frontend** | Vanilla JavaScript, HTML5, CSS3 |
| **Database** | MySQL |
| **AI/ML** | LangChain + Groq (LLaMA 3.3 70B), scikit-learn |
| **PDF Processing** | PyMuPDF |

## Architecture

```
Browser ──── Flask (app.py) ──┬── MySQL (professors, classes)
                              ├── JSON files (board tasks per class)
                              ├── CSV + scikit-learn (student data & grade prediction)
                              └── Groq LLM API (syllabus → tasks)
```

Flask serves both the frontend and the REST API on a single port. Professor accounts and class metadata live in MySQL. Each class gets its own JSON file for board persistence and its own student roster. The ML model takes three CIA scores as input and predicts the student's final end-semester letter grade.

## Grade Prediction Model

The grade prediction system is built on a custom synthetic dataset. I designed a probability distribution table based on real exam performance patterns observed at SASTRA University - mapping mark ranges to grade likelihoods across S, A+, A, B, C, D, and F. Noise was added to these distributions and ~9,000 synthetic student records were generated by sampling from them. A scikit-learn classifier was then trained on this dataset, taking three CIA (Continuous Internal Assessment) scores as input and predicting the final end-semester grade.

## Getting Started

### 1. Clone and install

```bash
git clone https://github.com/<your-username>/taskflow.git
cd taskflow
python -m venv .venv
source .venv/bin/activate   # Windows: .venv\Scripts\activate
pip install -r requirements.txt
```

### 2. Set up MySQL

```sql
CREATE DATABASE IF NOT EXISTS university;
USE university;

CREATE TABLE professors (
    id VARCHAR(50) PRIMARY KEY,
    name VARCHAR(100) NOT NULL,
    password_hash VARCHAR(255) NOT NULL
);

CREATE TABLE classes (
    id VARCHAR(50) PRIMARY KEY,
    professor_id VARCHAR(50) NOT NULL,
    name VARCHAR(100) NOT NULL,
    FOREIGN KEY (professor_id) REFERENCES professors(id)
);
```

### 3. Configure environment

```bash
cp .env.example .env
# Edit .env with your MySQL credentials and Groq API key (optional)
```

### 4. Run

```bash
python app.py
# Open http://localhost:5001
```

| `PUT` | `/api/students/:roll_no` | Update a student |
| `DELETE` | `/api/students/:roll_no` | Delete a student |
